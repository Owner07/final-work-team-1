# PFLB TEAM 1

## О проекте

Проект разработан в рамках дипломного задания по автоматизации тестирования и демонстрирует полный цикл автоматизации:

```text
UI → API → DB → Allure report → CI run
```

Цель проекта — реализовать автотесты для учебного полигона **PFLB Test-API** с проверками на трех уровнях:

* **UI** — проверка пользовательского интерфейса;
* **API** — проверка backend-методов;
* **DB** — проверка данных в PostgreSQL через JDBC.

---

## Что тестировали:

В приложении есть три основные сущности:

| Сущность | Описание     |
| -------- | ------------ |
| Users    | Пользователи |
| Cars     | Автомобили   |
| Houses   | Дома         |

Основные действия в системе:

* авторизация;
* создание пользователя;
* добавление денег пользователю;
* выдача кредита;
* создание автомобиля;
* покупка/продажа автомобиля;
* создание дома;
* заселение/выселение пользователя;

## Логирование

В проекте используется **log4j2**.

Логируются:

* старт теста;
* завершение теста;
* ошибки;
* действия в UI;
* API-запросы и ответы;
* SQL-запросы;
* результат DB-проверок.

Чувствительные данные не должны выводиться в логи.

## Требования к безопасности

В проекте нельзя хранить чувствительные данные напрямую в коде.

* логин приложения;
* пароль приложения;
* URL БД;
* логин БД;
* пароль БД.

Для CI используются **GitHub Secrets**.
